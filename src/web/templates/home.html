Why this pipeline

Security research, but with guardrails

Built for authorized OSS repositories. Every stage is sandboxed, every action is logged, and reports are never auto-submitted.

Pattern-driven recon

Semgrep + grep sweeps over the cloned repo to surface unsafe sinks: deserialization, eval/exec, SQL concatenation, and more.

Reasoning agents

Three model tiers — fast, reasoning, coder — with provider failover across Gemini, Groq, and OpenRouter.

Sandboxed PoCs

Generated proofs run in ephemeral Docker containers with --network none, read-only roots, capped CPU/RAM/PIDs.

Human-in-the-loop

Approval gates sit before Exploit and Report. Nothing weaponized, nothing disclosed without you in the loop.

Patch + regression test

Each finding ships with a unified diff and a regression test designed to fail before the fix and pass after.

HackerOne-ready reports

Structured Markdown reports with CVSS vector, reproduction steps, impact, and remediation — never auto-submitted.

Pipeline at a glance

Five stages, five typed agents

1

Recon

Clone the repo at a pinned ref, run pattern scans, and emit a ranked list of risky files and snippets.

2

Analyst

An LLM forms vulnerability hypotheses with CWE, severity, evidence, and a precise file/line reference.

3

Exploit

A non-destructive PoC is generated and validated in a Docker sandbox. Human gate before this stage.

4

Patch

The Coder tier proposes a minimal diff and a regression test that distinguishes vulnerable vs. patched behavior.

5

Report

A HackerOne-style Markdown report is written to disk with CVSS, summary, repro, impact, and remediation.
