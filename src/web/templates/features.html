Features

Five typed agents, three model tiers, one audited pipeline

Each stage is a typed Agent[TInput, TOutput] — its input and output are Pydantic models, so we can swap models, providers, and prompts without touching the orchestrator.

Pipeline stages

1

Recon — pattern sweep on a clean clone

Clones the target at a pinned ref, then runs grep + Semgrep patterns for unsafe deserialization, eval/exec, SQL string concatenation, command injection, weak crypto, and more. Outputs a ranked list of risky files and snippets.

2

Analyst — hypothesis generation

An LLM reads risky files in context and emits ranked vulnerability hypotheses. Each one carries a CWE, severity, file/line, evidence snippet, and a one-paragraph mechanism description — validated against a Pydantic model.

3

Exploit — non-destructive PoC + sandbox

The Coder tier writes a PoC script for the top hypothesis. We execute it inside Docker with --network none, a read-only root, and capped memory/CPU/PIDs. Validation is by sentinel string in stdout — never by side effect.

4

Patch — minimal diff + regression test

Proposes a unified diff against the cloned repo and a regression test that fails before the fix and passes after. The patch is never applied to the live repo — it lives as an artifact.

5

Report — HackerOne-style Markdown

Drafts a structured report with title, CVSS vector + score, summary, repro steps, impact, remediation, and references. Saved to disk as Markdown — never auto-submitted to a disclosure platform.

Platform features

Live progress

htmx-powered status panel polls every two seconds — stage chips, log tail, gate prompts, and artifacts appear as they're produced.

Token + quota tracking

Per-model daily request and token counts, with a UTC reset bar — so you never get blindsided by a hit quota mid-run.

Audit log viewer

Every event (run start, stage transition, gate decision, error) is appended to a SHA256 hash-chained log. The viewer flags broken chains.

Per-run artifacts

JSON for each stage plus a Markdown report — all rendered with stage-specific viewers (hypotheses, diffs, PoCs, etc.).

Provider failover

Gemini → Groq → OpenRouter chain with exponential backoff on 429/503. Per-model fallback handles individual quota exhaustion.

Findings index

SQLite-backed cross-run index. Filter by target; see severity, CWE, validation, patch, and report status at a glance.
