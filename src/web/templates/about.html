About

Security research that respects boundaries

Bug-Bounty Pipeline is an experimental platform that chains AI agents to identify, validate, and patch vulnerabilities in authorized open-source repositories. Every action is logged. No exploit is weaponized. No report is auto-disclosed.

What this project is

A reproducible research harness around five LLM-driven stages — Recon, Analyst, Exploit, Patch, Report — with provider failover, structured artifacts, and a SHA256 hash-chained audit log.

It's intentionally narrow in scope: we run only against repositories listed in config/targets.json. Adding a target requires an attestation that the user owns or has written permission to test the repo, and that attestation is recorded.

Safety design

ConcernMitigation
Targeting unauthorized reposAllowlist enforcement at submit time + attestation in audit log.
Weaponized exploitsExploit agent is system-prompted for non-destructive PoCs; destructive=true is refused.
Network exfiltration in PoCSandbox runs with --network none.
Host filesystem damageRead-only rootfs; workdir mounted :ro; capped --memory, --cpus, --pids-limit.
Auto-submission of reportsReport stage writes to disk only — no HTTP calls to disclosure platforms.
Audit tamperingEach entry SHA256-hashes the previous entry. audit verify checks the chain.

Limitations

  • No Docker in Replit — PoCs are generated and saved but cannot be executed there.
  • Gemini free-tier quotas reset daily at UTC midnight.
  • Each run currently exploits only the top-ranked hypothesis.
  • Patches are proposed diffs — they are never applied to the repo automatically.
